## Overview

OCX v1.4.0 introduces the **Unified Profile System**, replacing the legacy "ghost mode" configuration. This is a breaking change that requires migration.

## What Changed

| Before (v1.3.x) | After (v1.4.0) |
|-----------------|----------------|
| `ghost.jsonc` | `ocx.jsonc` |
| `.ghost/` directory | `.opencode/` directory |
| Ghost mode | Profile system |

### Requirements

- **OpenCode v1.1.29 or later** is required for profile system support.

## Migration Steps

There is no automatic migration command — v1.4.0 is a clean break from ghost mode. Follow the steps below to migrate manually.

### Step 1: Rename Config Files

```bash
# For each profile directory
for dir in ~/.config/opencode/profiles/*/; do
  if [ -f "${dir}ghost.jsonc" ]; then
    mv "${dir}ghost.jsonc" "${dir}ocx.jsonc"
  fi
done
```

### Step 2: Rename Local Config Directories

```bash
mv .ghost .opencode
```

### Step 3: Update References

Update any scripts or aliases that reference `ghost` commands.

## Verify Migration

After migrating, verify your setup:

```bash
# List profiles
ocx profile list --global

# Show merged config
ocx config show

# Launch OpenCode with a profile
ocx opencode -p default
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `ghost.jsonc` not found after migration | Migration already complete — `ocx.jsonc` should exist instead |
| Profile list empty | Run `ocx init --global` to reinitialize |
| OpenCode fails to launch | Verify OpenCode v1.1.29 or later is installed |
