## Overview

The KDCO Registry is a curated collection of OpenCode extensions deployed as a Cloudflare Worker. It serves components from `https://registry.kdco.dev`.

## Quick Start

```bash
# Install dependencies
bun install

# Build the registry
bun run build

# Local development
bun run dev

# Deploy to Cloudflare
bun run deploy
```

## Using the Registry

```bash
# Add a component directly
ocx add kdco/flow --from https://registry.kdco.dev

# Or add the registry permanently
ocx registry add https://registry.kdco.dev --name kdco
ocx add kdco/flow
```

## Available Components

### Bundles

| Name | Description |
|------|-------------|
| flow | Primary autonomous KDCO harness |
| workspace | Legacy/deprecated KDCO Workspace harness |
| philosophy | Code quality philosophies |

Migration pointer: new installs and profile defaults should target `kdco/flow`; keep `kdco/workspace` only for existing users or explicit legacy validation.

### Agents

| Name | Description |
|------|-------------|
| conductor | Read-only flow orchestrator |
| researcher | External research via MCP |
| explorer | Temp-root clone only; read/git-metadata/cleanup through dedicated tools; no package managers/interpreters/build/test/executable runs from clones |
| scribe | Documentation specialist |
| coder | Code implementation |
| plan-reviewer | Approves the saved plan before implementation |
| qa-reviewer | Approves the final result before commit/PR/report |

## Web Search Setup

The researcher agent uses **Exa** by default (free, no auth required). Additional search engines can be integrated via MCP servers.

### Adding Custom Search Engines

1. Configure the MCP server with command and environment
2. Store secrets securely using file-based or environment variables
3. Enable tools for the researcher agent using a glob pattern

**Preferred: File-based secrets:**

```bash
mkdir -p ~/.secrets && chmod 700 ~/.secrets
echo "your-api-key" > ~/.secrets/service-api-key
chmod 600 ~/.secrets/service-api-key
```

Reference in config with `{file:~/.secrets/service-api-key}`.
